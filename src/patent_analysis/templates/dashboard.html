Current Architecture

  • FastAPI backend with server-rendered Jinja2 frontend for fast localhost prototyping and later private hosting.
  • SQLite storage with a repository layer so we can later swap to PostgreSQL without changing the UI or service logic.
  • Rule-based feature extraction and scoring first, with OpenRouter as an optional enhancement path rather than a hard dependency.
  • Source adapters for demo JSON today and future live partner import folders or databases later.
